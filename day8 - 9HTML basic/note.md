What is Semantic HTML?
HTML tags that have a clear meaning or purpose.

They describe what the content is (e.g., <nav>), not just how it looks.

Semantic (Good): <article>, <header>, <table>

Non-Semantic (Styling only): <div>, <span>

Page Structure Tags
<header>: Top of page/section (logo, title, main menu).

<nav>: Main navigation links (menu bar).

<main>: The single, main content area of the page.

<footer>: Bottom of page/section (copyright, contact links).

<section>: A group of related content (e.g., "About Us" section).

<article>: Self-contained content (blog post, news story, comment).

<aside>: Sidebar content (related links, ads).

Text Content Tags
<h1> - <h6>: Headings (most to least important).

<p>: A paragraph of text.

<blockquote>: A long, multi-line quotation.

<strong>: Marks text as highly important (usually bold).

<em>: Stresses or emphasizes text (usually italic).

Other Common Tags
<figure>: A media element (like an image, chart).

<figcaption>: The caption for a <figure>.

<ul> / <ol> / <li>: Unordered list (bullets) / Ordered list (numbers) / List item.

<table>: For data in rows and columns.

<form>: A container for user input (buttons, text fields).
